About the Documentation

This User’s Guide is intended to help you navigate and use the web-based interface. This guide describes a variety of tasks, including many administrative tasks. It should be thought of as a guide to the UI for all users, including administrators at all levels (not just end users).

Note: This guide describes Nimbul’s “out-of-the-box” functions. If you have customized Nimbul, the descriptions and images may not match your installation.

Role/Access Level Tables

As you use this documentation, you will see tables that list topics by Nimbul role or access level. Example:

Role/Access Level Relevant Topics
Nimbul Administrators Managing User Accounts
All Users Signing Up for an Account
Permissions and Roles

In some tables, the phrase “and higher” is used: for example, a topic that is relevant for “Cluster Administrators (and higher)” should be read by Nimbul administrators, provider account administrators and cluster administrators. For descriptions of the various roles and access levels, see Permissions and Roles.

Individual topics also include “Permissions Required” tables where applicable.

Terms Used

To familiarize yourself with the terms used in this documentation, start by reviewing Navigation, UI Elements and the Glossary.

Further Reference

Because Nimbul is built on several Amazon Web Services APIs, you may find it useful to refer to the AWS API documentation.
